Phase 1: Creating Azure Account

To begin implementing SIEM in Microsoft Azure, follow these initial steps:

  • Sign up for a free Azure account with $200 worth of credits
  • Create a resource group named "HoneypotLab"
  • Navigate through Azure Portal for initial setup

Phase 2: Creating Windows Remote Server

Setting up the Virtual Machine:

  • Create a VM named "HoneypotVM"
  • Configure network settings and security groups
  • Allow all inbound traffic for monitoring purposes

Phase 3: Log Analytics Workspace

Essential steps for log analytics:

  • Create "LAHoneypot" workspace
  • Connect workspace to HoneypotVM
  • Configure workspace settings

Phase 4: Security Center Configuration

Setting up security monitoring:

  • Enable Defender for Servers
  • Configure data collection settings
  • Set up event collection parameters

Phase 5: Azure Sentinel Setup

Implementing SIEM solution:

  • Create Azure Sentinel instance
  • Connect to Log Analytics Workspace
  • Configure initial settings

Phase 6: VM Data Collection

Configuring the Virtual Machine:

  • Disable Windows Firewall
  • Set up PowerShell script for log collection
  • Configure ipgeolocation.io API integration

Phase 7: Custom Log Configuration

Setting up custom logging:

  • Create FailedRDPWithGeo_CL custom log
  • Configure field extraction
  • Set up parsing rules

Phase 8: Data Visualization

Creating visual representations:

  • Set up Azure Sentinel workbook
  • Configure world map visualization
  • Create custom queries for data analysis

Phase 9: Monitoring and Analysis

Ongoing maintenance and monitoring:

  • Regular workbook monitoring
  • Pattern analysis of attack data
  • Continuous visualization refinement

Implementation Time

2-3 hours for complete setup

Required Skills

Azure, PowerShell, Basic Networking
