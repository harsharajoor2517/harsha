IPv4 Subnetting

Gain hands-on experience with subnet masking, subnet calculation, and host calculation for IPv4 addresses.

IPv4 Subnet Mask Calculation

The CIDR notation /24 indicates that the first 24 bits are for the network portion. This results in a subnet mask of 255.255.255.0 in decimal format.

IPv4 Subnetting

To create 4 subnets, borrow 2 bits from the host portion. The new CIDR notation is /26.

  • Subnet 1: 10.10.0.0 with the range 10.10.0.0 - 10.10.0.63
  • Subnet 2: 10.10.0.64 with the range 10.10.0.64 - 10.10.0.127
  • Subnet 3: 10.10.0.128 with the range 10.10.0.128 - 10.10.0.191
  • Subnet 4: 10.10.0.192 with the range 10.10.0.192 - 10.10.0.255

IPv4 Host Calculation

For a network address 10.10.0.0 with a subnet mask 255.255.255.192 (CIDR /26), there are 6 bits available for host addresses. The number of usable hosts is \(2^6 - 2 = 62\).

IPv6 Subnetting

Gain hands-on experience with subnet masking, subnet calculation, and host calculation for IPv6 addresses.

IPv6 Subnet Mask Calculation

To create 16 subnets, borrow 4 bits from the host portion. The new CIDR notation is /52.

  • Subnet 1: 2001:db8::/52
  • Subnet 2: 2001:db8:1000::/52
  • Subnet 3: 2001:db8:2000::/52
  • Subnet 4: 2001:db8:3000::/52

IPv6 Host Calculation

For a subnet mask /64, there are 64 bits available for host addresses. The number of possible hosts is \(2^{64}\), approximately \(1.84 \times 10^{19}\).
