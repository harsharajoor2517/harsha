Project Overview

This project demonstrates the implementation of a Security Information and Event Management (SIEM) system using Microsoft Azure Sentinel to monitor and analyze RDP brute force attacks from around the world using a honeypot system.

Microsoft Azure Cloud

Microsoft Azure provides a robust cloud platform for implementing security solutions. Key components used in this project:

  • Virtual Machine deployment for honeypot setup
  • Log Analytics Workspace for data collection
  • Azure Sentinel for SIEM implementation
  • Real-time monitoring and visualization capabilities

Honeypot Implementation

A honeypot is created to deliberately attract potential attackers, allowing us to:

  • Monitor attack patterns and techniques
  • Collect data about attack origins
  • Understand common attack vectors
  • Test security measures in a controlled environment

RDP Failed LOG data Collection

The project utilizes a custom PowerShell script to:

  • Extract RDP failed logon logs from Windows Event Viewer
  • Parse out Windows Event Log information
  • Collect geographic information about attackers using ipgeolocation.io API
  • Generate custom logs with detailed attack data

Azure Sentinel

Azure Sentinel serves as the SIEM solution, providing:

  • Real-time threat detection and visualization
  • Geographic mapping of attack sources
  • Advanced analytics and machine learning capabilities
  • Automated incident response workflows

SIEM Advantages

Implementation of SIEM technology provides numerous benefits:

  • Centralized security event monitoring and management
  • Real-time threat detection and alerting
  • Automated incident response capabilities
  • Compliance reporting and audit trail maintenance
  • Enhanced visibility into security posture
